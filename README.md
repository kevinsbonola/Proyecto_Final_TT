# 📞 Análisis de Rendimiento de Operadores: CallMeMaybe

**¡Bienvenido a mi repositorio!** Este proyecto de análisis de datos fue desarrollado para auditar y optimizar las operaciones de **CallMeMaybe**, un servicio de telefonía virtual. 

## 🎯 Resumen del Proyecto
El objetivo principal de este análisis fue desarrollar un modelo basado en datos para identificar **operadores ineficaces** y descubrir áreas de mejora en la calidad del servicio al cliente. 

A partir de una base de datos con más de 45,000 registros, el proyecto se estructuró en las siguientes fases:
1. **Limpieza y Preprocesamiento (EDA):** Identificación y aislamiento de anomalías tecnológicas del sistema mediante la aplicación de filtros estadísticos (Percentil 99) para evaluar únicamente el rendimiento humano real.
2. **Ingeniería de Características:** Traducción de reglas de negocio abstractas en umbrales matemáticos estrictos (evaluando tiempos de espera, llamadas perdidas y volumen de salida).
3. **Pruebas de Hipótesis Estadísticas:** Uso de pruebas T de Welch para descubrir patrones de negocio, comprobando cómo el plan tarifario y el origen de la llamada afectan los tiempos de atención.
4. **Visualización Ejecutiva:** Creación de un Dashboard interactivo para la toma de decisiones gerenciales.

## 📊 Resultados Clave
* Se auditó a una plantilla de más de 1,000 operadores, identificando que el **56.5% mantiene una eficacia óptima**.
* Se detectó el principal cuello de botella operativo: **296 operadores** con un volumen deficiente de llamadas salientes.
* Se demostró estadísticamente que el sistema o los operadores dan menor prioridad de atención a las llamadas externas frente a las internas.
